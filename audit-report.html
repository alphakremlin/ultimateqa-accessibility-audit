Evidence Collector — Bug Report
Bug #1 — id="button1" Duplicated 5 Times
Critical
DetailThe attribute id="button1" appears on 5 separate elements. HTML IDs must be unique. getElementById("button1") returns only the first match. Breaks automation selectors and screen readers.
FixAssign unique IDs to each button: button1, button1-b, button1-c, etc.
Bug #2 — Radio Buttons Have No Programmatic Labels
Critical
DetailGender radio inputs (male/female/other) have no <label for="..."> association. Screen readers announce "radio button" with no label text. Clicking visible text does not toggle the radio.
FixAdd id to each input and link via <label for="gender-male">Male</label>
Bug #3 — Checkboxes Have No Programmatic Labels
Critical
DetailVehicle checkboxes (Bike/Car) have no label association. Screen reader users cannot identify what each checkbox represents.
FixAdd id to each checkbox and wrap with <label for="vehicle-bike">I have a bike</label>
Bug #4 — 4 Forms Using Invalid HTTP Method "LINK"
High
Detailmethod="LINK" is not a valid HTML form method. Only GET and POST are valid. Browsers silently fall back to GET — may produce unexpected behaviour.
FixReplace method="LINK" with method="GET" or method="POST".
Bug #5 — id="name" and id="email" Both Duplicated
High
DetailBoth id="name" and id="email" appear more than once on the same page. Breaks label associations and automation selectors.
FixEnsure all IDs are unique across the entire page.
Bug #6 — Select Dropdown Has No Label
High
DetailCar brand dropdown (Volvo/Saab/Opel/Audi) has no <label> or aria-label. Screen readers announce "combobox" with no context.
FixAdd <label for="car-select">Choose a car brand</label> and id="car-select" to the select element.
